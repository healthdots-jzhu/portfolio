## Key Changes in New Migration

### Table Structure
The `PortfolioVersions` table now uses:
- **ID Column**: `integer GENERATED BY DEFAULT AS IDENTITY` (auto-increment)
- **Previous**: `uuid` (GUID)
- **Benefits**: 75% storage reduction, faster queries, simpler debugging

### Unique Constraint
- **Constraint**: `(PortfolioId, VersionNumber)` UNIQUE
- **Index**: `IX_PortfolioVersions_PortfolioId_VersionNumber`
- **Purpose**: Prevents duplicate version numbers per portfolio (concurrency safety)

### Foreign Keys
- Portfolio → PortfolioVersions: CASCADE on delete
- User → PortfolioVersions: RESTRICT on delete (preserves audit trail)
